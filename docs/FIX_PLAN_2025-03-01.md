## 5. Validation Protocol (Short Cycle)
Run:
```bash
sudo .venv/bin/python src/bioxp/usb_driver.py
```

1. Reconnect and confirm board status at startup.
2. Motor:
   - quick status
   - motion interlock prep
   - one-click X/Z diagnostic
   - startup homing mimic
3. Chiller:
   - reachability
   - telemetry
   - baseline apply
   - hard reset recovery path
4. Thermal:
   - reachability
   - telemetry
   - baseline apply
   - hard reset recovery path
5. Repeat steps 2-4 for 10+ minutes to observe comm-drop behavior.

---

## 6. Exit Criteria for "Stable"
1. No unhandled no-response lockups during 10-minute mixed-operation run.
2. One-command recovery path exists for each major subsystem.
3. Motor, chiller, and thermal paths all expose clear status before and after control actions.
4. Documentation stays synchronized with menu behavior and control capabilities.
